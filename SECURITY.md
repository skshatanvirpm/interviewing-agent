# Security Policy

## Supported state

The current repository is suitable for local development and controlled demonstration. Public production deployment is not supported until authentication, authorization, rate limiting, Supabase RLS, and data-retention controls are implemented.

## Reporting a vulnerability

Report vulnerabilities privately to the project owner through a private GitHub security advisory after the repository is created. Until that channel exists, contact the owner directly through a private method.

Do not include the following in public issues:

- API keys or tokens;
- real resumes or transcripts;
- personal information;
- exploitable session identifiers;
- detailed reproduction steps for an unpatched vulnerability.

Include the affected component, impact, safe reproduction outline, and suggested mitigation where possible.
